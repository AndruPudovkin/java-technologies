                           -----DAO-----
На данный момент в проекте не подключена база данных.
Необходимо настроить liquibase, hibernate, spring data jpa. В качестве базы данных использовать PostreSQL.

Требования:
1. Класс User должен иметь поля: имя, фамилия, дата рождения, инн, снилс, номер паспорта, логин, пароль. Предусмотреть валидацию данных.
2. Информация о пользователях должна быть получена из базы данных.
3. Должны быть написаны интеграционные тесты для UserService с использованием тест контейнера postgresql.
4. Должен быть создан файл docker-compose для поднятия контейнера с БД для локального запуска.

                            -----Создать REST эндпоинт-----
Создать REST эндпоинт
Необходимо написать новый эндпоинт, который будет выдавать имя пользователя и номер счета клиента.

Для этого:
1. Необходимо создать сущность Requisites с полями: рассчетный счет, бик, корреспондентский счет, инн, кпп, кбк. Связь с сущностью пользователя - 1 к 1.
2. Создать эндпоинт, который будет получать имя пользователя, его рассчетный счет и кбк.

Требования:
1. Должны быть использованы Spring Data проекции для получения необходимой информации. Лишняя информация из базы данных кроме имени пользователя, рассчетного счета и кбк получена быть не должна.
2. Любое преобразование полученного объекта из базы данных запрещено. Объект должен быть отдан таким каким он был получен.
3. Должны быть написаны интеграционные тесты на написанный функционал
